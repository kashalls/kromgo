# Embedded assets

These files are embedded into the binary (`//go:embed`). They are vendored (not
fetched at runtime) so the service stays self-contained, works air-gapped, and
keeps a strict `script-src 'self'` Content-Security-Policy.

The gallery assets (`marked.min.js`, `github-markdown.css`, plus first-party
`gallery.css`/`gallery.js`) are served from `/assets/` for the index page. The icon
data (`mdi.txt.gz`) is embedded but **not** web-served; it is decoded into a
name→path map on first use.

| File                  | Upstream                                                                   | Version | License    | Source / regeneration                                                        |
| --------------------- | -------------------------------------------------------------------------- | ------- | ---------- | ---------------------------------------------------------------------------- |
| `marked.min.js`       | [marked](https://github.com/markedjs/marked)                               | 18.0.4  | MIT        | `https://cdn.jsdelivr.net/npm/marked@18.0.4/lib/marked.umd.min.js`           |
| `github-markdown.css` | [github-markdown-css](https://github.com/sindresorhus/github-markdown-css) | 5.9.0   | MIT        | `https://cdn.jsdelivr.net/npm/github-markdown-css@5.9.0/github-markdown.css` |
| `mdi.txt.gz`          | [@mdi/svg](https://github.com/Templarian/MaterialDesign-SVG)               | 7.4.47  | Apache-2.0 | generated by `cmd/genicons` (`mise run icons`) — gzipped `name\tpath` table  |

The jsDelivr `sourceMappingURL` comment is stripped from `marked.min.js` so the
file references nothing external. `mdi.txt.gz` holds every MDI glyph's 24×24 path
data, one `name<TAB>path` line per icon, gzipped; bump the version in
`cmd/genicons` and rerun `mise run icons` to update.

`gallery.css` and `gallery.js` are first-party (part of this repository).
